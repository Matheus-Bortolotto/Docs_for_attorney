📁 Como configurar o Google Drive
Siga os passos abaixo. Leva cerca de 5 minutos e é gratuito.
Passo 1
Acesse console.cloud.google.com com a conta Google do escritório e crie um novo projeto chamado "Juridoc".
Passo 2
No menu lateral: APIs e serviços → Biblioteca → busque "Google Drive API" → clique em Ativar.
Passo 3
Vá em APIs e serviços → Tela de consentimento OAuth → selecione Externo → preencha o nome do app e o e-mail → salve.
Passo 4
Vá em Credenciais → Criar credenciais → ID do cliente OAuth 2.0 → tipo: Aplicativo da Web → em "Origens JavaScript autorizadas" adicione http://localhost e o domínio do seu site → copie o Client ID gerado.
Passo 5
Cole o Client ID no campo da aba Integrações. No Google Drive, crie uma pasta chamada "Juridoc - Clientes", abra ela e copie o ID da URL (a parte após /folders/). Cole no campo "ID da pasta raiz".
🎯 Como configurar o Kommo
Siga os passos para obter o Access Token e os IDs do funil.
Passo 1 — Access Token
No Kommo: Configurações → Integrações → API. Clique em "Chaves de acesso" e gere um novo token. Copie o Access Token (o longo).
Passo 2 — Subdomínio
Olhe a URL da sua conta do Kommo. Se for escritorioadv.kommo.com, o subdomínio é escritorioadv.
Passo 3 — ID do funil e etapas
Acesse via API: https://SEU_SUBDOMINIO.kommo.com/api/v4/leads/pipelines com o token no header Authorization: Bearer SEU_TOKEN. O campo id é o ID do funil e dentro de _embedded.statuses estão os IDs de cada etapa.
Passo 4 — CORS (importante!)
O Kommo não permite chamadas diretas do navegador por CORS. Em produção, você vai precisar de um pequeno servidor (ex: Cloudflare Worker ou Vercel Function) que faça as chamadas pro Kommo. O portal já está pronto pra isso — basta trocar a URL da função nos campos acima. Fala comigo pra eu gerar o código do servidor →
